Performance testing

Testing Network Performance

So as we wrap up this chapter, we need ideally to have insight into how to assess network performance properly. Testing validates that the network is performing optimally and according to expectations. Using tools such as Ping, Traceroute, and Speed Test enables you to diagnose problems and ensure dependable connectivity. Such documentation and reporting of findings will provide support for ongoing network management and troubleshooting. In this last lesson, we will guide you through the critical parts that will safeguard and improve your network's reliability.

Step-by-Step Guide:

Step 1: Perform a Ping Test

Goal: Check if a device is reachable and measure response time.

  • Inspect Network Device: Open Command Prompt (Windows) or Terminal (Mac/Linux).
  • Type the following command:
    • ping [IP address or website]
    • Example: ping google.com
  • Press Enter.
  • Review the results:
    • Low time (ms) means a fast response.
    • High time or no response may indicate network issues.

Step 2: Run a Traceroute Test

Goal: Identify the path data takes across the network and detect any delays.

  • Open Command Prompt or Terminal.
  • Type the following command:
    • On Windows: tracert [IP address or website]
    • Example: tracert google.com
    • On Mac/Linux: traceroute [IP address or website]
    • Example: traceroute google.com
  • Press Enter.
  • Analyze the route and note if any specific hops (routers) are causing slowdowns.

Step 3: Conduct a Speed Test

Goal: Measure upload and download speeds to ensure they meet expectations.

  • Visit a reliable speed test website like Speedtest by Ookla.
  • Click Start Test.
  • Wait for the results to show your Download Speed, Upload Speed, and Ping.
  • Compare the results to your expected internet plan speeds.

Step 4: Conduct Connectivity Testing

Goal: Ensure all devices are connected and can access the internet.

  • Verify devices are connected to the network (Wi-Fi or Ethernet).
  • Try accessing multiple websites or online applications.
  • Check internal connectivity by pinging another device using its IP address:
    • ping [device IP address]
  • Ensure all devices communicate effectively within the network.

Documentation and Reporting

After testing, keeping detailed records is important for future troubleshooting and network management.

Here’s what to document:

  • Record Inspection Results:
    • Log all findings from the tests, including issues found and actions taken.
    • Helps track network performance over time.
  • Report Problems:
    • Inform the network administrator or IT team about any hardware failures, slow speeds, or security risks.
    • Quick reporting means faster solutions.
  • Schedule Regular Maintenance:
    • Set a schedule to test network performance at regular intervals.
    • Prevents potential problems before they affect users.
  • Keep Backup Records:
    • Save copies of network configurations, test results, and previous reports.
    • Useful when diagnosing future issues or during audits.

By following these steps, the network remains reliable, secure, and fast for all users.
